# RK/stats — трекер статистики Brawl Stars

PWA-приложение (ставится на телефон как обычное приложение) для просмотра статистики
игроков Brawl Stars. Работает на **официальном API Brawl Stars**, хостится на **Vercel**,
а историю/графики хранит в **Supabase** (опционально).

> Не аффилировано с Supercell. Иконки — из бесплатного CDN Brawlify (`cdn.brawlify.com`).

---

## Что умеет

**Из API напрямую (без базы):**
- Поиск игрока и клуба по тегу
- Профиль: трофеи, макс. трофеи, уровень, победы 3v3 / solo / duo, клуб
- Подсветка ника цветом (`nameColor`) и иконка профиля
- Все бойцы: power, ранг, трофеи, редкость (цвет полоски), сортировка
- Последние ~25 боёв: результат W/L/D, режим, карта, изменение трофеев, star player
- Win rate, «бои сегодня», star-player rate, разбивка по режимам и бойцам
- Вкладка **Ranked**: бои в ранкеде + бойцы, на которых играли
- **Сохранение** понравившихся матчей и профилей (в браузере)
- Клубы: значок, описание, трофеи, список участников
- Установка как приложение (PWA), офлайн-кэш иконок

**Через Supabase (вкладка History):**
- График трофеев во времени
- Таблица пуша кубков по дням
- Архив боёв (переживает лимит «последние 25»)

### Чего API НЕ даёт (важно, честно)
- **KD (киллы/смерти)** — API их не отдаёт. Показываем честный **Win Rate** и star-player rate.
- **Реплей боя с движением иконок по карте** — нет позиционных данных. Невозможно
  ни у нас, ни у Brawlify. Доступна только «карточка матча»: режим, карта, составы, результат.

## Шаг 1. Получить API-ключ Brawl Stars

1. Зайди на https://developer.brawlstars.com/ → войди (Supercell ID) → **Create New Key**.
2. В поле **Allowed IP Addresses** впиши IP прокси RoyaleAPI:
   ```
   45.79.218.79
   ```
   > Зачем: у Vercel «плавающие» IP, и ключ к ним не привязать. Поэтому запросы идут
   > через бесплатный прокси RoyaleAPI с фиксированным IP. Это стандартное решение.
3. Скопируй полученный токен — это `BRAWL_API_TOKEN`.

Базовый URL уже настроен на прокси: `BRAWL_API_BASE=https://proxy.royaleapi.dev/v1`.

---

## Шаг 2. Запуск на своём компьютере

Нужен **Node.js 18+** (https://nodejs.org).

```bash
# 1. распаковать проект и зайти в папку
cd rkstats

# 2. установить зависимости
npm install

# 3. создать файл .env из примера и вписать токен
cp .env.example .env
#   открой .env и впиши BRAWL_API_TOKEN=...

# 4. запустить
npm run dev
```

Открой http://localhost:3000 и введи тег, например `8PGPUQQG0`.

> Локально прокси тоже работает. Если хочешь без прокси — впиши свой домашний IP
> в whitelist ключа и поставь `BRAWL_API_BASE=https://api.brawlstars.com/v1`.

---

## Шаг 3. История и графики (Supabase) — опционально

Без этого шага приложение полностью работает, кроме вкладки **History**.

1. Создай проект на https://supabase.com (бесплатно).
2. **SQL Editor** → вставь содержимое `supabase/schema.sql` → **Run**.
3. **Project Settings → API** возьми:
   - `Project URL` → в `.env` как `NEXT_PUBLIC_SUPABASE_URL`
   - `service_role` ключ (секретный!) → как `SUPABASE_SERVICE_ROLE_KEY`
4. Придумай любую строку для `CRON_SECRET` (например `mysecret123`).

Как наполняется история: когда кто-то открывает вкладку History у игрока, этот игрок
добавляется в таблицу `tracked_players`. Дальше по расписанию (cron) сервер раз в час
делает снимок трофеев и подтягивает новые бои. График заполняется в течение следующих дней.

Проверить вручную можно так (после деплоя):
```
https://ТВОЙ-домен.vercel.app/api/cron/snapshot?secret=ТВОЙ_CRON_SECRET
```

---

## Шаг 4. Деплой на Vercel

1. Залей проект на GitHub (новый репозиторий).
2. https://vercel.com → **Add New → Project** → выбери репозиторий → **Import**.
3. В **Environment Variables** добавь всё из своего `.env`:
   - `BRAWL_API_TOKEN`
   - `BRAWL_API_BASE` = `https://proxy.royaleapi.dev/v1`
   - (если нужна история) `NEXT_PUBLIC_SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `CRON_SECRET`
4. **Deploy**.

Cron уже настроен в `vercel.json` (`/api/cron/snapshot` каждый час). На бесплатном плане
Vercel крон-запуски ограничены — если что, поменяй расписание на `0 0 * * *` (раз в сутки).
Vercel сам передаёт `Authorization: Bearer CRON_SECRET`, так что эндпоинт защищён.

## Шаг 5. Установить как приложение на телефон

Открой сайт в браузере телефона → меню → **«Добавить на главный экран»** /
**«Установить приложение»**. Появится иконка RK/stats, запуск — в полноэкранном режиме.

## Как поменять под себя

- **Название/цвета** — `app/globals.css` (переменные в `:root`) и компонент `components/TopBar.js`.
- **Иконка приложения** — замени `public/icons/icon.svg`, `icon-192.png`, `icon-512.png`.
- **Шрифт** — сейчас Lilita One (близок к шрифту Brawl Stars), подключён в `app/layout.js`.
- **Иконки бойцов/карт** — берутся из `cdn.brawlify.com` по официальному id, см. `lib/assets.js`.

## Структура проекта

```
app/
  layout.js              корневой layout, шрифты, регистрация PWA
  globals.css            дизайн (тёмная тема Brawl Stars)
  manifest.js            манифест PWA
  page.js                главная: поиск + сохранённые игроки
  player/[tag]/page.js   страница игрока (server-side fetch)
  club/[tag]/page.js     страница клуба
  api/
    player/[tag]/...      JSON-прокси игрока и боёв
    club/[tag]/route.js   JSON клуба
    history/[tag]/route.js  данные графика из Supabase
    cron/snapshot/route.js  снимок трофеев по расписанию
components/
  SearchBar, TopBar, SafeImg, SavedProfiles, PlayerView, TrophyChart
lib/
  bs.js        серверный клиент API (токен скрыт)
  assets.js    URL иконок, цвета редкости, форматирование, nameColor
  stats.js     аналитика по логу боёв (winrate, режимы, ранкед)
  supabase.js  серверный клиент Supabase (null, если не настроен)
  store.js     localStorage: сохранённые профили и матчи
supabase/schema.sql   таблицы для истории
vercel.json           расписание cron
```

## Возможные ошибки

- **403 / «API key rejected»** — токен неверный, либо в whitelist ключа не добавлен
  IP `45.79.218.79`. Проверь Шаг 1.
- **404 «Player not found»** — неверный тег. Тег без `#`, только буквы/цифры
  `0289PYLQGRJCUV`.
- **History пустая** — Supabase не настроен или cron ещё не отработал. Запусти эндпоинт
  cron вручную (Шаг 3) и подожди.
- **Иконки не грузятся** — у бойца/карты может не быть ассета в CDN; покажется пустое
  место (это нормально, есть fallback).
